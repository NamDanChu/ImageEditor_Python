#### **1. 프로젝트 개요 및 핵심 원칙**

*   **목표**: PyQt5나 Tkinter와 같은 외부 GUI 프레임워크 사용
*   **핵심 원칙**: 프로그램은 `while` 무한 루프를 기반으로 하는 **절차적 실행 구조**를 가진다. 매 루프마다 현재 UI(트랙바 등)의 상태를 읽어 이미지에 적용하고, 그 결과를 실시간으로 화면에 다시 그려주는 방식으로 동작한다.

### **2. 파일 구조에 맞춘 프로젝트 재설정**

먼저, 새로운 파일 구조가 각각 어떤 역할을 하는지 명확히 정의하겠습니다.

*   **`Final_ImageProcessing/`**: 프로젝트의 최상위 폴더(Root Directory)입니다. `main.py`를 실행할 때 현재 위치가 이곳이어야 합니다.

*   **`main.py`**: 프로그램의 총책임자(Orchestrator).
    *   UI(창, 트랙바, 가짜 버튼) 생성 및 관리.
    *   사용자 입력(마우스, 키보드) 감지.
    *   `image_processor` 패키지에 있는 적절한 함수를 **호출**하여 이미지 처리를 **요청**하는 역할.

*   **`image_processor/`**: 이미지 처리 기술자(Specialists)들이 모여있는 **패키지**.
    *   이 폴더는 단순한 폴더가 아니라, 파이썬 패키지(기능 묶음)입니다.
    *   **`__init__.py`**: 이 파일이 존재함으로써 파이썬은 `image_processor` 폴더를 하나의 import 가능한 패키지로 인식하게 됩니다. 이 파일을 통해 패키지의 함수들을 더 쉽게 불러올 수 있도록 설정할 수도 있습니다.
    *   **`pixel_processing.py`**: **화소(Pixel) 단위** 처리 전문가.
        *   픽셀 하나하나의 값을 바꾸는 기능들(그레이스케일, 밝기/명암 조절, 이진화 등)이 이곳에 들어갑니다.
    *   **`area_processing.py`**: **영역(Area)** 처리 전문가.
        *   주변 픽셀들을 함께 고려하는 기능들(블러, 샤프닝, 엣지 검출 등)이 이곳에 들어갑니다.
    *   **`geometric_processing.py`**: **기하학(Geometric)** 처리 전문가.
        *   이미지의 형태를 바꾸는 기능들(크기 조절, 회전, 대칭 등)이 이곳에 들어갑니다.
